Codeplay has a software platform called Acoran, that is designed to support C++ programming on a wide-range of hardware devices.  These devices include some RISC-V targets and simulators, in this project we would like to add support for an additional academic multi-core RISC-V simulator.  This project should allow us to test higher RISC-V core counts on a simulator than has previously been possible – this is of particular interest for the kind of massively-parallel programming models that Codeplay specializes in.  Also, this simulator uses 32-bit RISC-V cores, our previous RISC-V work has targeted 64-bit cores.

The Acoran platform includes an implementation of the heterogeneous C++ programming standard SYCL, ComputeCpp, and a toolkit for running that on new devices via OpenCL or Vulkan, ComputeAorta.  ComputeAorta does this by providing well-documented internal interfaces.  This project will involve porting those interfaces to the new RISC-V simulator; using existing test-suites to determine the completeness of the simulator and implementation pair; and creating a demonstration for the new port.

Responsibilities

  • Understand how ComputeAorta's internal interfaces work.
  • Port of boot code required to boot a program on the new simulator.
  • Create a new implementation of the simulator shim, tested on existing internal testsuites.
  • Run OpenCL and SYCL testsuites on top of simulator, documenting pass rates and areas of failures.  100% pass rate is not required inside this internship.
  • Choose some interesting demo application, e.g. SYCL-ONNX, that should work on the new simulator and resolve any issues stopping it from running.
  • Potentially, aid in improving our documentation for porting our software to new simulator or hardware targets.
